Please build this app step-by-step following these requirements:
1. Database Setup 
Create a schema.prisma file with two models:
* Purchase: id (String/UUID), date (DateTime, default now), description (String), amount (Float).
* Order: id (String, unique - this is the ManaPool order ID), date (DateTime), grossTotal (Float), platformFees (Float), netPayout (Float).
2. Backend Setup 
Set up an Express server (e.g., running on port 3001) with Prisma Client and the following routes:
* GET /api/dashboard: Calculate and return { totalExpenses: sum(Purchase.amount), totalRevenue: sum(Order.netPayout), netProfit: totalRevenue - totalExpenses }.
* GET /api/purchases: Return all purchases ordered by date descending.
* POST /api/purchases: Accept { description, amount, date } and create a new Purchase.
* GET /api/orders: Return all orders ordered by date descending.
* POST /api/manapool/sync:
    * Accepts { email, token } in the request body.
    * Makes a GET request to https://manapool.com/api/v1/seller/orders using headers: X-ManaPool-Email and X-ManaPool-Access-Token.
    * Parses the response. For each order, map it to our schema (grossTotal = total, platformFees = fees, netPayout = net_payout).
    * Use Prisma's upsert or createMany to save these into the Order table (ignore duplicates based on ID).
    * Return a success message with the count of new orders added.
Important: Configure vite.config.js to proxy /api requests to the Express server.

Create a clean, modern dashboard with a Sidebar navigation containing three tabs:
* Dashboard Tab: Display three large metric cards: Total Revenue (Green), Total Expenses (Red), and Net Profit (Blue/Green if positive, Red if negative).
* Purchases Tab (Money Out):
    * A form to add a new purchase (Inputs: Date, Description, Amount) and a submit button.
    * A table below displaying all historical purchases.
* Orders Tab (Money In):
    * A top panel with inputs for "ManaPool Email" and "ManaPool Token". Save these to localStorage so they persist.
    * A "Sync Orders from ManaPool" button that calls the /api/manapool/sync endpoint, shows a loading spinner, and refreshes the data upon success.
    * A table below displaying all synced orders (Date, Order ID, Gross, Fees, Net Payout).
